# asia-index-pages

## 项目简介

本仓库用于归档和发布多个独立的 HTML 页面。这些页面通常作为特定内容的索引页、导航页或信息页，以静态 HTML 文件的形式存储和分发。

仓库本身不针对任何域名或网站，仅作为一组 HTML 文件的集合地，便于版本管理与历史回溯。

## 目录说明

```
.
├── README.md          # 本说明文件
├── index.html         # 主索引页面（可选）
├── pages/             # 存放各独立 HTML 页面
│   ├── page1.html
│   ├── page2.html
│   └── ...
└── assets/            # 公共资源（CSS、图片、字体等）
    ├── css/
    ├── images/
    └── ...
```

- **pages/**：各独立 HTML 页面存放于此，每个文件对应一个独立的页面内容。
- **assets/**：存放页面共用的静态资源，按类型分目录组织。

## 页面归档说明

- 所有页面均为纯 HTML，不依赖后端服务，可直接通过浏览器打开。
- 页面间无强关联，各自独立，便于单独引用或部署。
- 归档内容遵循通用语义与结构，不包含指向特定网站的链接或营销信息。
- 页面文件名建议使用英文小写字母、数字与连字符，避免空格与特殊字符。

## 维护说明

- 新增页面：在 `pages/` 目录下创建新 HTML 文件，并确保其自包含（资源引用使用相对路径）。
- 更新页面：直接修改对应 HTML 文件，建议在提交信息中简要说明变更内容。
- 删除页面：移除对应文件，并确认无其他页面引用该资源。
- 资源管理：公共资源放入 `assets/`，避免重复；各页面独有资源可放在页面同级目录下。
- 提交规范：使用清晰简洁的提交信息，如“新增 XXX 页面”、“更新 XXX 页面样式”等。

## 其他

本仓库仅作为静态 HTML 文件的归档与发布用途，不提供动态服务或用户交互功能。如有特殊需求，请通过 Issue 或 Pull Request 提出。
